# Test Plan Review

审查测试方案的质量。

## 审查标准

1. **覆盖率**：测试用例是否覆盖所有模块接口
2. **边界条件**：是否包含正常、异常、边界值测试
3. **可执行性**：测试用例是否具体可执行（明确的输入和预期输出）
4. **分层合理性**：TDD 用例、集成测试、端到端测试划分是否合理
5. **TDD 就绪**：给实现阶段的用例是否足够驱动 TDD 循环

## 审查结论格式

```markdown
---
stage: test-plan
verdict: pass | fail | pass_with_notes
issues: ["问题 1", "问题 2"]
notes: ["建议 1"]
---
```
