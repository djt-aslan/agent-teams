# Architecture Review

审查系统架构设计的质量。

## 审查标准

1. **合理性**：模块划分是否职责清晰、边界合理
2. **可行性**：技术选型是否有充分理由
3. **完整性**：是否定义了模块间接口契约
4. **任务拆解**：实现任务是否拆解到位、耦合关系是否正确标注
5. **风险识别**：是否识别了关键技术风险和约束

## 审查结论格式

```markdown
---
stage: architecture
verdict: pass | fail | pass_with_notes
issues: ["问题 1", "问题 2"]
notes: ["建议 1"]
---
```
