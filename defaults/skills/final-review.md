# Final Review

流水线结束前的最终全面审查，从项目整体视角评估交付质量。

## 审查标准

1. **整体一致性**：各阶段产物之间是否一致（PRD ↔ 架构 ↔ 实现 ↔ 测试结果）
2. **交付完整性**：是否所有计划的功能都已实现并通过测试
3. **质量达标**：所有阶段的 Review 是否都已 pass
4. **遗留风险**：是否存在未解决的 issue 或技术债务
5. **可交付性**：代码是否准备好合并到主分支

## 审查结论格式

```markdown
---
stage: review
verdict: pass | fail | pass_with_notes
issues: ["问题 1", "问题 2"]
notes: ["建议 1"]
---
```
