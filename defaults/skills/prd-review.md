# PRD Review

审查产品需求文档的质量。

## 审查标准

1. **完整性**：是否覆盖所有功能需求和非功能需求
2. **清晰性**：每个用户故事是否有明确的验收标准
3. **优先级**：功能优先级划分是否合理
4. **一致性**：术语使用是否统一，无内部矛盾
5. **可衡量**：非功能指标是否量化（响应时间、并发量等）

## 审查结论格式

```markdown
---
stage: prd
verdict: pass | fail | pass_with_notes
issues: ["问题 1", "问题 2"]
notes: ["建议 1"]
---
```
