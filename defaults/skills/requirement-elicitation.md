# Requirement Elicitation

将用户的原始需求转化为结构化的需求文档。

## 输入
- 用户口头/文本需求描述

## 输出格式
- `artifacts/requirement.md`

## 要求
1. 列出所有功能需求，每条标明来源和优先级
2. 列出非功能性需求（性能、安全、可用性）
3. 标注不确定的需求点，供用户确认
4. 使用统一的术语，避免歧义
5. 不包含任何实现方案或技术选型

## 输出结构
```markdown
---
stage: requirement
status: completed
summary: "需求澄清完成，共识别 N 个功能需求和 M 个非功能需求"
---

# 需求文档

## 业务背景
## 功能需求
## 非功能需求
## 待确认项
```
