# Architecture Style Guide

## 设计原则
- **单一职责**：每个模块只做一件事
- **接口隔离**：模块间通过明确的接口通信，不直接依赖内部实现
- **依赖倒置**：高层模块不依赖低层模块，都依赖抽象
- **YAGNI**：不过度设计，不需要的抽象不留

## 文档要求
- **模块图**：用 ASCII 或 Mermaid 描述模块关系
- **接口契约**：每个对外接口标注：输入、输出、错误码、幂等性
- **数据流**：描述核心场景的数据流动路径

## 实现任务拆解要求
- 高耦合任务（修改同一模块）→ 同一批次，串行
- 无依赖任务（独立模块）→ 不同批次，可并行
- 每个批次包含的任务数不超过 5 个
