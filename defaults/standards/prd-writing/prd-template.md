# PRD Template

## 结构

### 1. 产品概述
- 产品愿景和价值主张
- 目标用户群体

### 2. 用户故事
```
作为 <角色>，我希望 <功能>，以便 <价值>
```
- 每个故事标注优先级（P0/P1/P2）

### 3. 功能详述
- 每个功能点的详细描述
- 验收标准（Given-When-Then 格式）
```
Given <前置条件>
When <用户操作>
Then <预期结果>
```

### 4. 非功能需求
- 性能指标（响应时间、并发量）
- 安全要求（认证方式、数据加密）
- 可用性要求

### 5. 优先级排期
- P0（必须交付）：...
- P1（重要）：...
- P2（可选）：...

_安装后按项目 PRD 规范调整。_
